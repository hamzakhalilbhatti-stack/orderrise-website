# OrderRise Stage 5.2 — Clear, Colorful 3D Communication

## Goal

Make every 3D scene understandable to a restaurant owner without requiring them to guess what the objects or animations represent.

## Color language

- Green — WhatsApp customer message
- Purple — AI understanding and classification
- Orange — Cart and checkout
- Amber — Kitchen preparation
- Blue — Inventory validation and stock
- Pink — Driver and delivery
- Cyan — Owner dashboard, reporting and completed outcome

## Homepage clarity improvements

- Seven numbered process stages above the hero scene
- Eight camera buttons: full view plus one for every stage
- Bright 3D labels that remain visible in front of geometry
- Color-matched floor zones and department markers
- Active stage highlighting
- Written explanation beside every interactive demo action
- Input, action and output explanation for every scroll journey stage
- Purpose panels for product pillars, devices and Admin Hub modules
- Matching color legend between the website interface and 3D scene

## Subpage clarity improvements

Every Product, Features, How It Works, Demo, Pricing, About, Contact and solution page includes:

- A visible scene explanation overlay
- A numbered process rail
- Bright labels and process colors
- Local Three.js files instead of remote CDN dependence

## Testing

Automated checks cover:

- JavaScript syntax
- Three.js module bundle resolution
- 20 HTML pages
- 8 restaurant solution pages
- Local file references
- Duplicate HTML IDs
- Import maps
- All JavaScript DOM IDs
- Local HTTP delivery of every project file
- Presence of all clarity controls and explanation panels

Use `diagnostics.html` on your own computer to verify WebGL rendering and animation.
